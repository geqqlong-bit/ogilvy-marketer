---
name: mc-compete
description: 竞品情报分析技能。接收竞品信息（结构化表格、自然语言或文件），自动去重、分类、评估威胁等级，输出差异化应对建议。
---

## 触发条件

当用户提到以下任意情形时使用本技能：

- "竞品分析" / "竞品动态" / "对手在做什么"
- "帮我看看这几个竞品"
- 提供了竞品相关的信息（文字、表格、文件）
- "对比一下我们和 XX 的打法"

---

## 工作方式

1. 接收竞品信息，支持三种输入格式：
   - **结构化表格**：`竞品名 | 平台 | 动作 | 信息角度 | 可攻击点`
   - **自然语言描述**：系统自动提取结构
   - **文件导入**：CSV / TSV / TXT，自动解析
2. 读取 `brief.md`（如存在）获取产品定位和人群信息
3. 去重 → 分类 → 威胁评估 → 应对建议
4. 写入 `campaigns/{project-slug}/compete.md`

---

## 处理流程

### 1. 去重

按 `竞品名 + 平台 + 动作` 组合去重，保留信息最完整的条目。

### 2. 分类

根据动作描述中的关键词自动归类：

| 类别 | 触发关键词 |
|------|-----------|
| 价格促销 | 低价、折扣、优惠、大促、买赠、满减 |
| 达人种草 | 达人、KOL、KOC、测评、探店、试用 |
| 直播转化 | 直播、连播、直播间、秒杀 |
| 付费投放 | 投流、投放、冲量、DOU+、信息流、竞价 |
| 品牌叙事 | 成分、功效、品牌故事、理念、联名 |
| 内容铺量 | （以上均不匹配时） |

### 3. 威胁评估

每个竞品动作评估：

- **直接威胁**：正在争夺同一人群 + 同一平台 + 同一心智位置
- **间接威胁**：同一品类但不同定位或不同平台
- **可借力**：竞品动作创造了品类热度，可以借势

### 4. 应对建议

每个竞品动作给出具体应对：

```markdown
### {竞品名} · {平台}

**动作：** {具体动作}
**角度：** {信息角度}
**类别：** {分类}
**威胁等级：** 直接威胁 / 间接威胁 / 可借力

**可攻击点：** {弱点}
**应对建议：**
- {具体怎么打，不是"做差异化"，是"用 XX 证据攻击 XX 弱点"}

**执行优先级：** 高 / 中 / 低
```

---

## 交互规则

- 用户可以随时追加竞品信息，系统合并去重后更新 compete.md
- 用户可以问"XX 竞品最近在干什么"，系统基于已有数据回答
- 如果有 brief.md，应对建议会结合我方产品定位；如果没有，给出通用建议
- 竞品情报可以独立使用，不依赖完整 campaign 流程

---
