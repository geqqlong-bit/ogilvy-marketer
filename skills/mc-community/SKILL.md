## 为什么需要社群/私域

在公域平台（抖音/小红书/TikTok），品牌不拥有用户，平台拥有。算法变了，流量就没了。

私域/社群解决的核心问题：

1. **流量资产私有化** — 用户在你的社群里，不被平台算法绑架
2. **复购成本趋近于零** — 触达已有用户几乎免费
3. **用户洞察直接获取** — 真实反馈不需要经过平台数据筛选
4. **品牌关系深度化** — 从"买过"变成"关系"

## 工作方式

1. 确认社群策略范围：
   - **平台**：微信私域 / Discord / WhatsApp / Telegram / 线下社群
   - **品牌阶段**：从 0 搭建 / 已有基础想激活 / 成熟体系想优化
   - **目标**：拉新+沉淀 / 激活+复购 / 用户洞察+共创 / 裂变增长
2. 读取 `brand.md`（品牌人格 → 社群氛围定义）
3. 读取 `storyteller.md`（叙事体系 → 社群内容调性）
4. 产出写入 `campaigns/{project-slug}/community.md`

## 模块 1：社群定位与架构

### 社群类型选择

| 社群类型 | 核心价值 | 适合品类 | 运营重点 |
|---------|---------|---------|---------|
| 产品服务群 | 售后+答疑+使用指导 | 高客单/复杂产品 | 解决问题的响应速度 |
| 兴趣/生活方式群 | 话题讨论+内容分享 | 生活方式品牌 | 内容质量和氛围 |
| 会员专属群 | VIP 权益+优先信息 | 有复购的品类 | 特权感和尊享感 |
| 学习/成长群 | 知识分享+打卡 | 教育/健康/效率 | 持续价值输出 |
| 共创群 | 新品内测+用户反馈 | 创新型品牌 | 让用户有参与感和影响力 |
| 裂变群 | 拉新+分销+任务 | 高频低客单 | 激励机制设计 |

## 模块 2：用户分层运营

### RFM 分层模型

| 层级 | 定义 | 运营策略 |
|------|------|---------|
| **超级用户** | 高频购买+主动推荐+深度互动 | 1v1 关系维护，共创邀请，品牌大使 |
| **活跃用户** | 定期购买+参与讨论 | 专属福利，新品优先体验 |
| **普通用户** | 偶尔购买+偶尔浏览 | 激活活动，场景化推送 |
| **沉默用户** | 入群后不活跃 | 召回活动，价值重新展示 |
| **流失用户** | 超过 N 天无互动/无购买 | 流失原因调研+召回策略 |

### 用户标签体系

```markdown
## 用户标签体系

### 基础标签
- 入群渠道：{电商包裹卡/社媒引流/线下活动/朋友推荐}
- 首购产品：
- 累计消费：
- 入群时间：

### 行为标签
- 互动频率：{高活跃/中等/低活跃/沉默}
- 内容偏好：{产品讨论/生活分享/优惠信息/知识干货}
- 购买偏好：{新品尝鲜/促销敏感/常规复购}

### 价值标签
- 用户价值等级：{S/A/B/C}
- 推荐意愿：{高/中/低}
- 共创参与度：{积极/偶尔/从不}

### 标签使用规则
- 标签不是贴上就不变的，定期（月/季）刷新
- 关键行为触发标签升降级（如购买后自动升级）
- 运营动作根据标签组合触发（如"高活跃+未复购" → 推专属优惠）
```

### 企业微信运营

**添加好友策略：**
- 包裹卡引流（"扫码加专属顾问，领取 XX 福利"）
- 公众号菜单栏/关注后自动推送
- 直播间引导加微
- 线下门店引导

**朋友圈人设打造：**
| 内容类型 | 占比 | 目的 |
|---------|------|------|
| 产品/品牌内容 | 30% | 种草+信任 |
| 生活方式/人设 | 40% | 建立真实感 |
| 用户反馈/案例 | 15% | 社会证明 |
| 促销/活动 | 15% | 转化 |

**SOP 节点（新用户 0-30 天）：**

```markdown
## 新用户私域 SOP

| 时间 | 动作 | 内容 | 目标 |
|------|------|------|------|
| Day 0 | 欢迎语 | 自我介绍+入群引导+首单福利 | 建立第一印象 |
| Day 1 | 使用指引 | 产品使用方法/注意事项 | 降低退货率 |
| Day 3 | 关怀询问 | "用得怎么样？有什么问题随时问我" | 建立关系 |
| Day 7 | 内容推送 | 使用技巧/搭配建议 | 提升使用频率 |
| Day 14 | 调研邀请 | "帮我们做个小调研，送 XX" | 收集反馈 |
| Day 21 | 专属优惠 | 针对未复购用户的专属价 | 推动复购 |
| Day 30 | 会员邀请 | "成为 VIP 会员，享受 XX" | 长期关系 |
```

### 微信社群运营

**社群内容排期（周维度）：**

| 星期 | 内容主题 | 形式 | 目的 |
|------|---------|------|------|
| 一 | 产品知识/使用技巧 | 图文/短视频 | 价值输出 |
| 二 | 用户晒单/分享日 | 用户 UGC | 社区氛围 |
| 三 | 话题讨论 | 抛话题引导 | 互动活跃 |
| 四 | 新品/活动预告 | 图文 | 期待感 |
| 五 | 限时福利/秒杀 | 小程序 | 转化 |
| 六日 | 轻松内容/生活方式 | 图文/视频 | 陪伴感 |

**社群活跃度维护：**

| 问题 | 解法 |
|------|------|
| 群里没人说话 | 管理员每天固定时间抛话题；设置打卡/签到机制 |
| 只有广告没有讨论 | 控制广告频率（每天最多 1 条）；增加话题内容占比 |
| 有人发广告/负面 | 群规前置+违规提醒+严重者移除 |
| 用户投诉在群里 | 立刻私聊处理，群内简短回应"已经在处理" |
| 人数增长停滞 | 老带新裂变活动+包裹卡持续引流 |

**适用品牌：** 面向 Gen Z / 游戏/潮流/科技/Web3/DTC

```markdown
- #feedback — 用户反馈/建议

- #share-your-look — 用户晒单（适合美妆/服饰）

### 会员专属频道（需要角色权限）
- #vip-lounge — 会员专属讨论
- #early-access — 新品提前看
- #exclusive-deals — 专属优惠

### 机器人设置
- 新人欢迎 bot
- 等级/经验值系统（活跃奖励）
- 投票/调研 bot
```

### WhatsApp / Telegram

**适用市场：** 东南亚、中东、欧洲、拉美

**运营要点：**
- WhatsApp Business API：自动化消息 + 产品目录
- Telegram 群组：大规模社群（上万人）、频道+群组双轨
- 消息频率：每周 2-3 条，不要过度打扰
- 核心价值：独家优惠 + 新品抢先 + 客服快速响应

```markdown

|------|------|------|---------|
```

## 交互规则

- 社群策略可以独立使用，也可以作为 campaign 的一个环节
- 用户可以说"我有 3 个微信群但活跃度很低"，系统诊断并给激活方案
- 用户可以说"帮我从头搭一个 DTC 品牌的 Discord"，系统输出完整架构
- 运营 SOP 可以直接交给运营团队执行
- 输出完成后告诉用户文件路径，不要在对话中重复全文
