## 触发条件

当用户提到以下任意情形时使用本技能：

- 新品上市 / 品牌推广 / 节日大促 / 内容矩阵 / 全案
- 需要"一套方案" / "完整 campaign" / "作战计划" / "执行方案"
- 用自然语言描述了项目背景、产品、人群、平台中的任意两项以上
- 跨境出海 / 海外推广 / global launch / DTC campaign

---

## 工作方式

本技能在 OpenClaw 对话中运行。你拥有文件读写能力。

- 所有 campaign 产出写入 `campaigns/{project-slug}/` 目录
- `project-slug` 由项目名生成（小写、连字符、去特殊字符）
- 每个产出是一个独立 markdown 文件，方便单独分发
- 避免空话和模板套话

### 语言与市场规则

- **纯国内 campaign**：全程中文
- **纯海外 campaign**：策略和渠道文档用中文（给操盘手看），内容产出用目标市场语言（给执行团队和用户看）
- **跨境双线 campaign**：国内平台内容中文，海外平台内容目标语言，策略/渠道/审核用中文
- 用户可以随时指定语言偏好

## 第一步：需求分诊

### 必须确认的字段

| 字段 | 缺失影响 | 缺失时处理 |
|------|----------|-----------|
| 产品/项目名 | 所有产出缺主语 | 必须追问 |
| 核心目标 | 策略无法聚焦 | 必须追问 |
| 目标人群 | 内容和平台失准 | 必须追问 |
| 主平台 | 无法出平台资产 | 必须追问 |
| 目标市场 | 合规体系和语言无法确定 | 从平台推断：国内平台→中国市场，海外平台→对应市场 |
| 预算量级 | KPI 无法落地 | 可做假设并标注 |
| 活动窗口 | 节奏无法排布 | 可做假设并标注 |
| 品牌调性 | 审校无参照 | 可做假设并标注 |
| 风险边界 | 合规审查无标准 | 可做假设并标注 |

**规则：**
- 前 4 个字段缺失时，只追问缺失项，不输出任何方案
- 后 4 个字段缺失时，做合理假设并用 `⚠️ 假设` 标注，不停下
- 如果用户说"直接给方案"，对所有缺失字段做假设并标注

### 产出：brief.md

确认完字段后，将结构化 brief 写入 `campaigns/{project-slug}/brief.md`，格式：

```markdown
# {项目名} · 作战 Brief

**创建时间：** {当前日期}

## 基本信息
- **产品/服务：**
- **项目代号：**
- **核心目标：**
- **KPI：**
- **预算：**
- **活动窗口：**
- **重点区域：**

## 人群
- **目标人群：**
- **人群痛点：** （从目标和产品推断）
- **决策阻力：** （从品类和人群推断）

## 品牌
- **品牌调性：**
- **核心卖点：** （列 2-4 个）
- **表达禁区：**

## 阵地
- **目标市场：** （中国 / 美国 / 欧洲 / 日本 / 东南亚 / 跨境多市场）
- **主阵地：**
- **辅助阵地：**
- **渠道约束：**

## 竞品
| 竞品 | 阵地 | 动作 | 角度 | 可攻击点 |
|------|------|------|------|----------|

## 风险边界
-

## 交付要求
-
```

## 第二步：策略规划（策略负责人视角）

读取 `brief.md`，输出策略总纲。

### 必须包含

1. **核心传播主张** — 一句话定位，不是口号，是"为什么选你"的理由
2. **信息屋** — 三层结构：主张 → 支撑证据 → 人群钩子
3. **竞品差异化** — 逐个竞品分析弱点，给出具体打法（不是"差异化定位"这种空话）
4. **阶段节奏** — 按活动窗口拆分：预热 / 引爆 / 收口，每阶段的核心动作和成功标志
5. **预算分配假设** — 按 内容生产 / 达人合作 / 付费推广 / 弹性预算 四层拆分百分比

### 竞品分析分类规则

根据竞品动作关键词自动归类：
- 含"低价/折扣/优惠/大促/买赠" → 价格促销
- 含"达人/KOL/KOC/测评/探店" → 达人种草
- 含"直播/连播/直播间" → 直播转化
- 含"投流/投放/冲量/DOU+/信息流" → 付费投放
- 含"成分/功效/品牌/理念/故事" → 品牌叙事
- 其他 → 内容铺量

### 产出：strategy.md

写入 `campaigns/{project-slug}/strategy.md`。

---

## 第三步：内容生产（内容策划视角）

读取 `brief.md` + `strategy.md`，为每个平台输出内容矩阵。

### 每个平台必须输出

1. **3 个选题角度**，每个角度含：
   - 标题钩子样例（不是"参考方向"，是可以直接用的标题）
   - 推荐内容形态（图文/短视频/长文/直播切片）
   - 开头第一句或脚本前 3 秒
2. **内容调性指引** — 结合品牌调性，给出这个平台的具体表达要求
3. **CTA 设计** — 每个角度对应的转化动作

### 平台适配规则

**国内平台：**

| 平台 | 内容重点 |
|------|---------|
| 小红书 | 真实体验、标题钩子、封面视觉、搜索关键词 |
| 抖音 | 前 3 秒钩子、强节奏脚本、口播感、转化动作 |
| 微信 | 长文结构、私域承接、社群转化、信任建立 |
| 微博 | 热点挂钩、话题性、舆情扩散 |
| B站 | 测评深度、故事完整度、评论区互动 |
| 私域 | 社群话术、企微承接、留资钩子 |

**海外平台：**

| 平台 | 内容重点 |
|------|---------|
| Instagram | 视觉优先、Reels 短视频、Carousel 信息密度、Hashtag 策略 |
| TikTok | 前 1-2 秒 hook、trending sounds、文字叠层、Spark Ads |
| YouTube | 缩略图 + 标题 CTR、Shorts 短视频、长视频 SEO 和章节结构 |
| Facebook | Feed 首行可见区、Video 静音友好、Groups 社群运营 |
| X (Twitter) | 280 字符钩子、Thread 深度叙事、引用转推策略 |
| Pinterest | 竖版视觉 SEO、Board 规划、Idea Pin 教程内容 |
| LinkedIn | 专业洞察、Carousel PDF、行业观点型钩子 |
| Amazon | 产品标题 SEO、五点描述、A+ Content、Review 引导 |
| Google Ads | RSA 标题矩阵、Display 文案、Landing Page 对齐 |
| Email | Subject Line A/B 变体、Welcome 序列、CTA 按钮文案 |

### 产出

每个平台一个文件，写入 `campaigns/{project-slug}/content/{平台名}.md`。

---

## 第四步：渠道排布（渠道投放视角）

读取 `brief.md` + `strategy.md`，输出渠道执行计划。

### 必须包含

1. **主阵地节奏** — 预热期 / 引爆期 / 收口期的具体动作（不是"加大投放"，是"D-7 铺 5 篇 KOC 笔记覆盖 3 个核心搜索词"）
2. **辅助阵地联动** — 复用规则、格式适配、承接逻辑
3. **达人策略** — 量级建议（KOC 为主还是 KOL 为主）、频次、素材复用
4. **投放动作** — 资源位、AB 测试方案、优化信号定义
5. **约束回应** — 必须明确回应 brief 中的渠道约束
6. **监测节奏** — KPI checkpoint 设置（日/周维度）

### 产出：channel.md

写入 `campaigns/{project-slug}/channel.md`。

---

## 第五步：品牌 + 合规双审（品牌审校 + 合规审查视角）

读取 campaign 目录下所有已产出的文件，执行双审。

### 品牌审校

- 检查全部产出是否符合 brief 中的品牌调性
- 检查 CTA 是否有过度销售感（与品牌定调冲突）
- 检查核心卖点是否被充分支撑（用细节证明，不是直接宣称）
- 给出 **通过 / 需修改** 结论，附具体修改项

### 合规审查

- **国内市场**：检查广告法风险（绝对化用语、功效承诺、虚假原价、医疗暗示）+ 平台规则
- **美国市场**：检查 FTC Endorsement Guides（付费披露、功效声明、虚假证言）+ 平台规则
- **英国市场**：检查 ASA/CAP Code（广告标识、误导声明、HFSS 限制）
- **欧盟市场**：检查 GDPR（数据收集、cookie 同意）+ EU 广告规则 + Green Claims Directive
- **日本市场**：检查景品表示法 + ステマ規制 + 薬機法
- **跨境 campaign**：各目标市场法规分别审查
- 对照 brief 中的风险边界
- 给出 **通过 / 需修改** 结论，附具体风险项和修改建议

### 门禁逻辑

- 如果品牌或合规任一项为"需修改"，在总结中标记 `🚫 未达上线标准`
- 如果全部通过，标记 `✅ 可上线`
- 无论是否通过，都附加声明：`所有生成内容仅供参考，上线前请由具备资质的法务或合规人员复核。`

### 产出：review.md

写入 `campaigns/{project-slug}/review.md`，包含品牌审校和合规审查两个章节。

---

## 第六步：作战总览

所有步骤完成后，输出一个总览摘要到对话中（不写文件），包括：

1. **campaign 目录结构** — 列出所有已生成的文件
2. **一句话策略** — 从 strategy.md 提取核心主张
3. **平台覆盖** — 列出已生成内容的平台
4. **审核状态** — 品牌/合规是否通过
5. **下一步建议** — 如有"需修改"项，指出优先修改什么

---

## 交互规则

- **增量执行**：用户可以在任意步骤后说"先到这里"，系统停在当前步骤，已产出的文件保留
- **单步重跑**：用户可以说"策略重新来一版"或"小红书内容重做"，系统只重跑对应步骤
- **补充信息**：用户可以随时补充竞品、调整人群、修改预算，系统更新 brief.md 并提示哪些下游产出需要刷新
- **不要复述**：不要在对话中大段复述已写入文件的内容，告诉用户文件路径即可
- **文件即交付**：每个 markdown 文件都应该可以直接发给团队成员使用，不需要二次编辑格式
