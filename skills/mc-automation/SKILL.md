## 与其他技能的关系

| | mc-automation | mc-content | mc-campaign |
|---|---|---|---|
| 关注点 | 系统架构：什么时候、发什么、给谁 | 内容本身：写什么、怎么写 | 策略全局：目标、节奏、渠道 |
| 产出 | 自动化蓝图、工作流规格、评分模型 | 文案、脚本、内容矩阵 | 策略总纲、brief、渠道计划 |
| 协作 | 向 mc-content 要内容素材，向 mc-campaign 要策略上下文 | 按 mc-automation 的触发场景产出对应内容 | 在渠道计划中引用自动化蓝图 |

## 工作方式

1. 确定自动化目标：
   - **业务场景**：新客获取 / 线索培育 / 购后运营 / 流失召回 / 忠诚度管理
   - **渠道组合**：Email / SMS / Push / 企微 / WhatsApp / In-app / 广告重定向
   - **工具环境**：用户已有或计划使用的 MarTech 工具（可不指定，产出工具无关蓝图）
2. 读取 `brief.md`（如存在）获取人群和产品上下文
3. 产出写入 `campaigns/{project-slug}/automation.md`

### 核心原则

- **工具无关**：产出的是自动化逻辑和规格，不是某个工具的操作教程
- **可直接落地**：工程师/运营拿到蓝图就能在对应工具中配置，不需要再翻译
- **写出触发条件和分支逻辑**：不是"发一封邮件"，是"当用户完成注册后 5 分钟，如果未完成首单，发送 Email A；如果已完成首单，发送 Email B"

## 模块 1：客户生命周期映射

### 生命周期阶段定义

```markdown
## 客户生命周期

### 阶段定义

| 阶段 | 定义 | 进入条件 | 退出条件 | 核心指标 |
|------|------|---------|---------|---------|
| 匿名访客 | 未识别身份的网站/内容访客 | 首次触达 | 留资/注册 | 访问量、内容互动 |
| 已知线索 (Lead) | 已留资但未激活 | 提交表单/关注/注册 | 完成激活动作 | 留资转化率 |
| 激活用户 (MQL) | 表现出购买意向的线索 | 达到评分阈值 | 首购/流失 | MQL→SQL 转化率 |
| 首购客户 | 完成首次交易 | 首单支付 | 复购/沉默 | 首购 AOV、复购率 |
| 活跃客户 | 有持续购买/互动行为 | 复购 | 沉默超 X 天 | LTV、复购频次 |
| 沉默客户 | 超过 X 天无互动 | 最后互动超时 | 回归/流失 | 召回率 |
| 流失客户 | 超过 Y 天无互动 | 沉默超时 | 召回 | 挽回率 |

### 生命周期流转图
{用文字描述或 ASCII 图表示阶段间的流转路径和关键触发点}
```

### 阶段参数需要用户确认

| 参数 | 说明 | 默认值（如用户未指定） |
|------|------|---------------------|
| 激活动作 | 什么行为算"激活" | 首次加购 / 首次深度浏览 |
| 沉默阈值 | 多久不互动算"沉默" | 30 天（电商）/ 14 天（SaaS）|
| 流失阈值 | 多久不互动算"流失" | 90 天（电商）/ 60 天（SaaS）|
| 高价值客户定义 | 什么条件算 VIP | 累计消费前 10% / 复购 ≥3 次 |

## 模块 2：自动化工作流设计

### 工作流规格格式

每个工作流必须按以下格式输出：

```markdown
### 工作流：{名称}

**目标：** {这个工作流要解决什么问题}
**触发条件：** {什么事件启动这个工作流}
**退出条件：** {什么条件下用户退出这个工作流}
**频控：** {同一用户在 X 天内最多触发 Y 次}
**优先级：** {当多个工作流同时触发时的优先级}

#### 流程

1. **触发：** {事件描述}
   ↓
2. **等待：** {时长} | **条件判断：** {条件}
   ↓ 是 / ↓ 否
3. **动作（是）：** {渠道} · {内容摘要} · {CTA}
4. **动作（否）：** {渠道} · {内容摘要} · {CTA}
   ↓
5. **等待：** {时长} | **条件判断：** {条件}
   ...

#### 所需内容素材
| 步骤 | 渠道 | 内容需求 | 状态 |
|------|------|---------|------|

#### 度量
| 指标 | 目标值 | 追踪方式 |
|------|--------|---------|
```

### 标准工作流模板库

以下为常见场景的工作流框架，根据用户业务场景选择并定制：

#### A. 新用户欢迎序列 (Welcome Series)

```
触发：用户注册/关注/留资
↓
[即时] 发送欢迎消息（品牌介绍 + 首单利益点）
↓ 等待 24h
[D1] 条件判断：是否已浏览商品？
  → 是：发送浏览商品相关的内容/推荐
  → 否：发送品牌故事 + 热门商品推荐
↓ 等待 48h
[D3] 条件判断：是否已加购/收藏？
  → 是：发送限时首单优惠
  → 否：发送社会证明（用户评价/案例）
↓ 等待 48h
[D5] 条件判断：是否已首购？
  → 是：转入"购后跟进"工作流 → 退出
  → 否：发送最后机会提醒（优惠即将过期）
↓ 等待 72h
[D8] 未转化 → 降低触达频率，转入常规培育
```

#### B. 购物车/浏览放弃挽回 (Abandonment Recovery)

```
触发：加购/深度浏览后未下单，离开超过 1 小时
↓
[+1h] 发送提醒（商品图 + "你忘了什么"）
↓ 等待 24h
[+24h] 条件判断：是否已购买？
  → 是：退出
  → 否：发送社会证明或库存紧迫感
↓ 等待 48h
[+72h] 条件判断：是否已购买？
  → 是：退出
  → 否：发送优惠激励（如有预算）
↓
退出。不再追。3 次未转化不继续打扰。
```

#### C. 购后跟进 (Post-Purchase)

```
触发：订单支付成功
↓
[即时] 订单确认 + 物流预期
↓
[送达+1天] 使用指南 / 开箱引导
↓
[送达+5天] 满意度调查（NPS / 简单评分）
  → 高分：引导评价/晒单 + 推荐奖励
  → 低分：触发客服跟进工作流
↓
[送达+14天] 关联商品推荐 / 复购提醒
↓
[送达+30天] 条件判断：是否复购？
  → 是：转入"活跃客户"工作流
  → 否：转入"沉默预警"工作流
```

#### D. 流失召回 (Win-Back)

```
触发：用户进入"沉默"阶段
↓
[第1天] 品牌关怀（"好久没见你"+ 近期新品/内容）
↓ 等待 7 天
[第8天] 条件判断：是否回归互动？
  → 是：退出，恢复常规沟通
  → 否：发送专属召回优惠
↓ 等待 14 天
[第22天] 条件判断：是否回归互动？
  → 是：退出
  → 否：最后一次触达（"我们会想你的" + 取消订阅选项）
↓
标记为"流失"，停止主动触达。
```

#### E. VIP/忠诚度管理

```
触发：用户达到 VIP 条件
↓
[即时] VIP 欢迎 + 专属权益说明
↓
[持续] 按月：专属内容/早鸟通知/限量款预告
↓
[降级预警] 近 X 天消费低于阈值 → 提醒维持资格
↓
[升级] 达到更高层级 → 升级通知 + 新权益
```

#### F. 事件驱动型 (Event-Triggered)

```
触发条件库（按业务场景选择）：
- 生日/纪念日 → 祝福 + 专属优惠
- 浏览特定品类 X 次 → 品类专属推荐
- 优惠券即将过期 → 过期提醒
- 价格变动 → 关注商品降价通知
- 补货周期到期 → 复购提醒（适合消耗品）
- 活动开始前 X 天 → 预热通知
```

## 模块 3：Lead Scoring 模型

### 评分维度

```markdown
## Lead Scoring 模型

### 行为评分（Behavioral Score）

| 行为 | 分值 | 衰减规则 |
|------|------|---------|
| 访问网站/落地页 | +2 | 每次，日上限 10 分 |
| 浏览产品详情页 | +5 | 每次 |
| 加入购物车 | +15 | 每次 |
| 下载资料/白皮书 | +10 | 每次 |
| 参加 Webinar/直播 | +20 | 每次 |
| 申请 Demo/试用 | +30 | 每次 |
| 打开邮件 | +1 | 每次，日上限 3 分 |
| 点击邮件链接 | +3 | 每次 |
| 社媒互动（点赞/评论） | +2 | 每次 |
| 主动咨询（客服/表单） | +25 | 每次 |
| 连续 7 天无互动 | -10 | 每周 |
| 连续 30 天无互动 | -30 | 一次性 |
| 取消订阅 | -50 | 一次性 |

### 属性评分（Demographic/Firmographic Score）

| 属性 | 加分条件 | 分值 |
|------|---------|------|
| 职位/角色 | 决策者/预算持有者 | +20 |
| 公司规模 | 匹配 ICP 区间 | +15 |
| 行业 | 目标行业 | +10 |
| 地理位置 | 目标市场 | +5 |
| 预算确认 | 明确表达预算意向 | +25 |

### 评分阈值与动作

| 分数区间 | 阶段 | 自动化动作 |
|---------|------|-----------|
| 0-30 | Cold Lead | 内容培育序列 |
| 31-60 | Warm Lead | 加速培育 + 个性化推荐 |
| 61-80 | MQL | 通知销售团队 + 发送 Case Study |
| 81-100 | SQL | 销售直接跟进 + Demo 预约 |
| 100+ | Hot Lead | 即时通知 + 优先响应 |
```

**注意：** 以上为框架参考值。分值和阈值必须根据用户的业务数据校准——先上线运行 2-4 周，再根据实际转化数据调整。

## 模块 4：多渠道编排

### 渠道选择矩阵

```markdown
## 渠道编排策略

### 渠道能力与适用场景

| 渠道 | 到达率 | 时效性 | 内容丰富度 | 成本 | 最佳场景 |
|------|--------|--------|-----------|------|---------|
| Email | 中 | 低 | 高 | 极低 | 内容推送、培育序列、促销通知 |
| SMS | 高 | 高 | 低 | 中 | 紧急通知、验证码、限时促销 |
| Push 通知 | 中 | 高 | 低 | 极低 | App 用户召回、实时提醒 |
| 企业微信 | 高（中国） | 高 | 高 | 低 | 1v1 服务、社群运营、私域转化 |
| WhatsApp | 高（海外） | 高 | 中 | 中 | 客服、订单通知、1v1 营销 |
| In-app 消息 | 高 | 即时 | 中 | 极低 | 功能引导、交叉推荐 |
| 广告重定向 | 中 | 低 | 中 | 高 | 冷用户召回、品牌强化 |

### 渠道优先级规则

用户可接触多个渠道时的选择逻辑：
1. 优先使用用户最近互动的渠道
2. 紧急/时效性消息：SMS > Push > 企微/WhatsApp > Email
3. 内容型消息：Email > 企微/WhatsApp > In-app > Push
4. 同一消息不在多个渠道重复发送
5. 用户如果在渠道 A 已完成目标动作，取消渠道 B 的排队消息

### 频控规则

| 维度 | 规则 |
|------|------|
| 单渠道日上限 | Email ≤2 / SMS ≤1 / Push ≤3 / 企微 ≤3 |
| 跨渠道日上限 | 所有渠道合计 ≤5 条消息 |
| 营销类周上限 | ≤3 次主动营销触达 |
| 静默时段 | 22:00-08:00 不发送（除紧急事务通知） |
| 退订尊重 | 退订后立即停止该渠道所有营销消息 |
```

## 模块 5：度量与优化

### 自动化效果指标体系

```markdown
## 自动化度量体系

### 工作流级指标

| 工作流 | 进入量 | 完成率 | 核心转化率 | 贡献收入 |
|--------|--------|--------|-----------|---------|

### 渠道级指标

| 渠道 | 送达率 | 打开率 | 点击率 | 转化率 | 退订率 |
|------|--------|--------|--------|--------|--------|

### 健康度指标

| 指标 | 健康阈值 | 预警阈值 | 当前值 |
|------|---------|---------|--------|
| Email 送达率 | >95% | <90% | |
| Email 打开率 | >20% | <10% | |
| SMS 送达率 | >98% | <95% | |
| Push 点击率 | >3% | <1% | |
| 退订率 | <0.5%/月 | >1%/月 | |
| 垃圾投诉率 | <0.1% | >0.3% | |

### A/B 测试框架

每个工作流至少应有以下测试：
| 测试维度 | 变量 | 度量指标 |
|---------|------|---------|
| 发送时间 | 即时 vs 延迟 X 小时 | 打开率 |
| 内容类型 | 促销 vs 内容 vs 社会证明 | 点击率 |
| CTA | 不同按钮文案 | 转化率 |
| 渠道 | Email vs SMS vs Push | 转化率 + 成本 |
| 频次 | 3 步 vs 5 步序列 | 最终转化率 + 退订率 |
```

## 场景适配

### 电商 / DTC

重点工作流：Welcome → Abandonment → Post-Purchase → Win-Back → VIP
重点渠道：Email + SMS + Push（海外）/ 企微 + SMS（国内）
重点评分：行为评分为主，RFM 分层

### B2B / SaaS

重点工作流：Lead Nurture → MQL→SQL Handoff → Onboarding → Expansion
重点渠道：Email + In-app + LinkedIn
重点评分：属性评分 + 行为评分并重，需要 Sales + Marketing 对齐

### 私域 / 社群

重点工作流：加粉 → 欢迎 → 内容培育 → 活动转化 → 裂变
重点渠道：企微 + 社群 + 朋友圈 + 小程序
重点评分：互动行为评分，社群活跃度

## 交互规则

- 用户可以指定场景（"帮我设计购后跟进流程"），系统只输出对应工作流
- 用户可以指定工具（"我们用 Klaviyo"），系统在蓝图中标注工具特有的实现要点
- 用户可以指定渠道（"我们只用 Email 和企微"），系统只覆盖指定渠道
- 如果有 brief.md，自动化蓝图会对齐 campaign 的人群、平台和目标
- 内容素材需求清单可以直接交给 mc-content 生产对应内容
- 输出完成后告诉用户文件路径，不要在对话中重复全文
