## Storyteller 的角色定义

mc-content 回答"怎么写"，mc-storyteller 回答"讲什么故事，为什么讲这个故事"。

一个好的 Storyteller 做三件事：

1. **定义冲突** — 没有冲突就没有故事。品牌存在是因为它在对抗什么。
2. **分配角色** — 用户是英雄还是品牌是英雄？品牌在用户故事里扮演什么？
3. **设计弧线** — 用户从"认识"到"购买"到"推荐"的情感旅程，是一条设计过的故事弧线。

---

## 工作方式

1. 确认叙事范围：
   - **层级**：品牌层叙事（永恒的品牌故事）/ Campaign 层叙事（阶段性传播主线）/ 单条内容叙事
   - **输入**：品牌策略（brand.md）、文化洞察（insight.md）、目标人群
2. 读取 `brand.md`（必须——品牌人格和定位是叙事的起点）
3. 读取 `insight.md`（如存在——文化环境决定什么故事有共鸣）
4. 产出写入 `campaigns/{project-slug}/storyteller.md`

## 模块 1：核心冲突定义

每个伟大的品牌都在对抗一个明确的"敌人"。这个敌人不是竞品——是一种现状、一种偏见、一种用户痛苦的根源。

### 冲突框架

**品牌对抗的"旧世界"**
在我们存在之前，用户面对的是什么糟糕的现状？

**品牌承诺的"新世界"**
如果我们成功了，用户的世界会变成什么样？

**冲突的张力**
旧世界和新世界之间的张力越大，故事越有力量。

```markdown
## 核心冲突

### 旧世界（我们对抗的现状）
{用一段话描述用户在没有这个品牌时面对的困境/不公/不爽}

### 新世界（我们创造的可能）
{用一段话描述这个品牌想要实现的改变}

### 核心张力
{一句话概括冲突：如 "XX 不应该意味着 XX"}

### 冲突验证
- 这个冲突对目标用户是真实的吗？（不是品牌自嗨）
- 这个冲突是持久的还是短期的？（品牌故事需要长期张力）
- 竞品有没有占据类似的冲突？（需要差异化）
```

### 经典品牌冲突参考

| 品牌 | 对抗的旧世界 | 承诺的新世界 |
|------|-----------|-----------|
| Nike | "你不够好，你做不到" | "Just Do It — 每个人都是运动员" |
| Apple | "科技是冰冷的、复杂的" | "科技可以是美的、人性的" |
| Dove | "美是狭隘的单一标准" | "真实的美，每个人都美" |
| Patagonia | "消费主义在破坏地球" | "商业可以拯救地球" |

## 模块 2：叙事角色分配

品牌故事里有两个核心角色：用户和品牌。谁是主角？

### 叙事角色模型

**模型 A：用户是英雄，品牌是导师**（StoryBrand 框架）

```
用户（英雄）遇到了一个问题
   ↓
品牌（导师）出现并提供方案
   ↓
品牌给出清晰的行动指引
   ↓
用户采取行动
   ↓
用户获得成功（避免了失败）
```

适合：大多数消费品牌，尤其是工具型/解决方案型产品

**模型 B：品牌是主角，用户是追随者**

```
品牌有一个信念/使命
   ↓
品牌在践行这个信念的路上做了什么
   ↓
用户被品牌的信念吸引，加入这个阵营
```

适合：使命驱动型品牌（Patagonia）、文化品牌（Supreme）、领袖型品牌

**模型 C：用户是主角，品牌是同伴**

```
用户有一个自我成长的旅程
   ↓
品牌陪伴用户走过这段旅程
   ↓
品牌和用户之间形成情感纽带
```

适合：生活方式品牌、长期关系型品牌（保险、教育、健康）

```markdown
## 叙事角色

### 选定模型：{模型 A/B/C}
理由：{为什么这个角色分配最适合我们的品牌和用户关系}

### 品牌的角色
- 身份：{导师/领袖/伙伴/守护者}
- 品牌说话的姿态：{俯视/平视/仰视 — 答案几乎永远应该是平视}
- 品牌在用户生活中的存在感：{关键时刻出现 / 日常陪伴 / 幕后推动}

### 用户的角色
- 用户在故事里是谁：{正在成长的人 / 已经很好但可以更好的人 / 在寻找答案的人}
- 用户想成为什么样的人：{品牌帮助他/她靠近这个身份}
- 用户害怕什么：{品牌帮他/她避免这个恐惧}
```

## 模块 3：品牌故事母题

母题是反复出现的叙事元素，让用户在不同触点看到品牌时感到"这就是他们的风格"。

### 母题类型

**情感母题**（品牌反复唤起的核心情感）

| 母题 | 含义 | 品牌例子 |
|------|------|---------|
| 发现 (Discovery) | 找到新的可能性 | Airbnb / GoPro |
| 归属 (Belonging) | 找到自己的位置 | Lululemon / 泡泡玛特 |
| 蜕变 (Transformation) | 变成更好的自己 | Nike / Peloton |
| 自由 (Freedom) | 打破束缚 | Harley-Davidson / 元气森林 |
| 守护 (Protection) | 被安全地包裹 | Volvo / 无印良品 |
| 仪式 (Ritual) | 日常中的特别时刻 | 星巴克 / Aesop |
| 真实 (Authenticity) | 做真正的自己 | Dove / Glossier |
| 掌控 (Mastery) | 驾驭能力的满足感 | Apple / Dyson |

**视觉母题**（品牌反复出现的视觉符号）
- 一个重复的色彩组合
- 一个标志性的构图方式
- 一种质感（磨砂/光泽/手工感）
- 一个符号元素（如 Aesop 的药剂瓶、Apple 的留白）

**语言母题**（品牌反复出现的措辞模式）
- 一个标志性的句式结构
- 一个反复出现的关键词
- 一种叙事节奏（短句急促 / 长句舒缓 / 问答式）

```markdown
## 品牌故事母题

### 情感母题
- 主母题：{母题名} — {为什么选它}
- 辅母题：{母题名} — {在什么场景使用}

### 视觉母题
- {符号/元素 1}：{在什么场景出现}
- {符号/元素 2}：{在什么场景出现}

### 语言母题
- 标志性句式：{举例}
- 反复出现的词：{举例}
- 叙事节奏：{描述}
```

## 模块 4：情感弧线设计

从用户第一次接触品牌到成为忠实用户，是一条设计过的情感旅程。

### 情感弧线地图

```markdown
## 用户情感弧线

### 阶段 1：认知（Awareness）
- 用户状态：不知道品牌存在
- 情感目标：好奇 / 被吸引 / "这是什么？"
- 叙事策略：{用冲突吸引注意力 / 用视觉冲击建立第一印象}
- 典型触点：信息流广告 / 小红书种草 / TikTok 偶遇

### 阶段 2：兴趣（Interest）
- 用户状态：被吸引，开始主动了解
- 情感目标：认同 / 共鸣 / "这个品牌懂我"
- 叙事策略：{展示品牌价值观 / 讲用户自己的故事}
- 典型触点：品牌主页 / 产品详情页 / KOL 深度内容

### 阶段 3：信任（Trust）
- 用户状态：认同价值观，但不确定产品
- 情感目标：安心 / 确认 / "它真的可以"
- 叙事策略：{社会证明 / 真实用户故事 / 透明展示}
- 典型触点：用户评价 / UGC 内容 / 试用体验

### 阶段 4：转化（Conversion）
- 用户状态：准备购买
- 情感目标：果断 / 期待 / "我值得拥有"
- 叙事策略：{降低决策门槛 / 创造紧迫感 / 未来自我投射}
- 典型触点：购买页面 / 限时活动 / 开箱预期

### 阶段 5：体验（Experience）
- 用户状态：收到并使用产品
- 情感目标：惊喜 / 超预期 / "选对了"
- 叙事策略：{开箱仪式感 / 产品内彩蛋 / 售后关怀}
- 典型触点：包装 / 产品本身 / 售后邮件

### 阶段 6：拥护（Advocacy）
- 用户状态：满意，愿意推荐
- 情感目标：骄傲 / 归属 / "我想让别人也知道"
- 叙事策略：{赋予用户表达工具 / 创造值得分享的时刻 / 社区归属感}
- 典型触点：分享内容 / 社群 / 复购仪式
```

## 模块 5：平台叙事适配

同一个品牌故事，在不同平台用不同的叙事结构讲。

### 各平台叙事逻辑

```markdown
## 平台叙事适配

### 小红书
- 叙事结构：发现体 / 安利体 / 干货清单
- 叙事视角：第一人称"我"的体验分享
- 情感锚点：{与品牌相关的特定情感}
- 故事长度：800-1500字 或 图文组合
- 禁忌：硬广感、品牌方口吻、说教

### 抖音 / TikTok
- 叙事结构：反转钩子（3秒hook → 制造认知冲突 → 揭示答案）
- 叙事视角：第一人称 / 第三人称旁观
- 情感锚点：{惊讶/共情/满足}
- 故事长度：15-60秒（信息密度极高）
- 禁忌：铺垫太长，前3秒不抓人

### Instagram
- 叙事结构：美学叙事（视觉先行，文字辅助）
- 叙事视角：品牌视角的生活方式展示
- 情感锚点：{向往感/品味认同}
- 故事长度：视觉为主，Caption 150字以内
- 禁忌：过度推销、低质量视觉

### 品牌官网 / 独立站
- 叙事结构：英雄旅程（品牌创始故事 / About页 / 品牌宣言）
- 叙事视角：品牌全知视角
- 情感锚点：{信任/使命感/品质}
- 故事长度：完整叙事，可以较长
- 禁忌：自说自话不和用户关联

### 微信公众号 / 长文
- 叙事结构：问题-洞察-方案-行动
- 叙事视角：专家/朋友/观察者
- 情感锚点：{认同/启发}
- 故事长度：2000-4000字
- 禁忌：纯品牌吹嘘，无用户价值

### 广告（付费投放）
- 叙事结构：痛点-方案-证据-行动（PASE）
- 叙事视角：最快建立共鸣的视角
- 情感锚点：{紧迫感/期待感}
- 故事长度：视频 6-30秒 / 图文一屏信息
- 禁忌：故事太长，CTA不清晰
```

## 模块 7：故事库（Story Bank）

品牌需要一个可以反复取用的故事素材库。

```markdown
## 品牌故事库

### 起源故事（Origin Story）
{品牌从何而来？创始人为什么做这件事？触发点是什么？}

### 用户故事模板（User Stories）
每个真实用户故事都应该包含：
1. 背景：{这个人是谁，过着什么样的生活}
2. 困境：{TA 遇到了什么问题}
3. 转折：{TA 如何发现/使用了我们的产品}
4. 改变：{TA 的生活/感受发生了什么变化}
5. 洞察：{这个故事揭示了什么关于人性/品牌/品类的真相}

### 产品故事（Product Stories）
每个核心产品背后：
- 为什么做这个产品（起因）
- 做的过程中最难的决策是什么（过程）
- 最终给用户带来的改变是什么（结果）

### 观点故事（POV Stories）
品牌对行业/趋势/议题的看法：
- 我们相信 {XX}
- 我们不同意 {XX}
- 我们发现了 {XX}
```

## 输出模式

### 快速叙事定向

用户说"品牌故事怎么讲"时，输出：
- 核心冲突（一句话）
- 角色分配（品牌 vs 用户）
- 情感母题
- 控制在 1 页以内

### 完整叙事体系

用户要"系统做一套品牌叙事"时，输出全部 7 个模块。

### Campaign 叙事

用户说"这个 campaign 的故事线怎么设计"时，重点输出模块 6。

## 交互规则

- mc-storyteller 需要 brand.md 作为输入，如不存在会提示先做 mc-brand
- 可以独立使用（"帮我设计品牌故事"）
- 也可以在 mc-campaign 之前使用（为 campaign 设计叙事线）
- 用户可以说"这个 campaign 用什么故事结构"，系统给出适配的叙事方案
- 输出完成后告诉用户文件路径，不要在对话中重复全文
