## 触发条件

当用户提到以下任意情形时使用本技能：

- "品牌策略" / "品牌定位" / "品牌战略" / "brand strategy"
- "我们品牌是谁" / "品牌核心价值" / "品牌使命" / "brand purpose"
- "品牌人格" / "品牌调性" / "品牌声音" / "tone of voice"
- "品牌架构" / "主品牌和子品牌的关系" / "brand architecture"
- "品牌升级" / "品牌焕新" / "rebranding"
- "从零开始建一个品牌" / "品牌怎么定义"
- "所有内容风格不统一" / "品牌感觉很模糊"

---

## 为什么 mc-brand 是最上游的技能

```
mc-brand（我们是谁）
  ↓
mc-storyteller（我们怎么讲故事）
  ↓
mc-insight（文化环境告诉我们什么值得讲）
  ↓
mc-campaign / mc-content / mc-aigc / mc-kol ...（执行层）
```

没有 brand.md，其他所有技能都在猜"这个品牌应该用什么语气说话"。有了 brand.md，所有下游技能自动对齐。

---

## 工作方式

1. 确认品牌策略范围：
   - **起点**：从零创建品牌 / 已有品牌需要梳理 / 品牌焕新升级
   - **输入**：创始人理念、产品特性、目标市场、竞品格局
   - **深度**：核心定位（快速版）/ 完整品牌圣经
2. 读取 `brief.md`、`research.md`、`insight.md`、`compete.md`（如存在）作为输入
3. **核心产出写入 `campaigns/{project-slug}/brand.md`** — 这个文件是全局品牌上下文，其他所有技能会读取它

## 模块 1：品牌本质（Brand Essence）

品牌本质是一切品牌决策的原点——一个品牌存在的根本理由。

### 品牌本质金字塔

```
                    ┌─────────┐
                    │ 品牌使命  │  ← 我们为什么存在
                    │ Purpose  │
                    ├─────────┤
                 ┌──│ 品牌愿景  │──┐  ← 我们想创造什么世界
                 │  │ Vision   │  │
                 ├──┼─────────┼──┤
              ┌──│  │ 品牌价值观 │  │──┐  ← 我们相信什么
              │  │  │ Values   │  │  │
              ├──┼──┼─────────┼──┼──┤
           ┌──│  │  │ 品牌承诺  │  │  │──┐  ← 我们对用户的核心承诺
           │  │  │  │ Promise  │  │  │  │
           └──┴──┴──┴─────────┴──┴──┴──┘
```

```markdown
## 品牌本质

### 品牌使命（Purpose）
{一句话：这个品牌存在是为了什么}

WHY — 创始人为什么要做这件事（不是"赚钱"，是内心的那个驱动力）
{创始人故事/初心的一句话提炼}

### 品牌愿景（Vision）
{如果这个品牌成功了，世界会有什么不同？}

### 品牌价值观（Values）
不是挂在墙上的标语，而是真正影响日常决策的原则。

| 价值观 | 含义 | 在实际决策中意味着什么 |
|--------|------|---------------------|
| {价值观 1} | {解释} | {遇到 XX 情况时，我们选择 XX 而不是 XX} |
| {价值观 2} | | |
| {价值观 3} | | |

测试标准：如果一个价值观不能帮你做出一个困难的取舍，它就是空话。

### 品牌承诺（Promise）
{用户选择我们而不是竞品，我们承诺给他/她什么}
- 功能性承诺：{产品层面}
- 情感性承诺：{感受层面}
- 社会性承诺：{身份/群体层面，如适用}
```

## 模块 2：品牌定位（Brand Positioning）

定位不是"我们想成为什么"，是"在用户心智中，我们占据哪个唯一的位置"。

### 定位公式

```
对于 {目标用户}，
{品牌名} 是 {品类/赛道} 中
唯一一个 {差异化核心} 的品牌，
因为 {支撑理由 / RTB}。
```

### 定位地图

```markdown
## 品牌定位

### 定位陈述
{完整定位公式}

### 竞争参照框架
| 维度 | 我们 | 竞品 A | 竞品 B | 竞品 C |
|------|------|--------|--------|--------|
| 核心卖点 | | | | |
| 价格带 | | | | |
| 目标人群 | | | | |
| 品牌调性 | | | | |
| 渠道重心 | | | | |

### 定位选择的逻辑
- 为什么选这个位置（而不是其他）：{空白大 / 能力匹配 / 文化趋势支撑}
- 这个位置的风险：{可能被挑战的地方}
- 防御策略：{如何巩固这个位置}

### 品类入口词
当用户想到 {什么场景/需求} 时，第一个想到我们。
- 主入口：{核心关联场景}
- 副入口：{次要关联场景}
```

## 模块 3：品牌人格（Brand Personality）

如果品牌是一个人，TA 是谁？

### 人格原型

从 12 种经典品牌原型中选择最匹配的 1-2 个：

| 原型 | 核心动机 | 品牌例子 | 适合品类 |
|------|---------|---------|---------|
| 天真者 (Innocent) | 追求简单、纯粹、幸福 | 可口可乐、多芬 | 食品/个护/儿童 |
| 探索者 (Explorer) | 自由、发现、突破边界 | North Face、Jeep | 户外/旅行/科技 |
| 智者 (Sage) | 知识、真理、洞察 | Google、BBC | 教育/科技/咨询 |
| 英雄 (Hero) | 勇气、掌控、证明自己 | Nike、Under Armour | 运动/健康/工具 |
| 叛逆者 (Outlaw) | 打破规则、颠覆传统 | Harley-Davidson、Diesel | 潮流/科技/亚文化 |
| 魔法师 (Magician) | 转化、创造奇迹 | Apple、Dyson | 科技/美妆/高端 |
| 普通人 (Everyman) | 归属感、真实、接地气 | IKEA、优衣库 | 大众消费品 |
| 情人 (Lover) | 亲密、美感、愉悦感官 | Chanel、Godiva | 奢品/美妆/食品 |
| 小丑 (Jester) | 快乐、幽默、享受当下 | M&M's、老干妈 | 食品/娱乐/年轻品牌 |
| 照顾者 (Caregiver) | 保护、关怀、利他 | Johnson's、Volvo | 母婴/健康/保险 |
| 创造者 (Creator) | 创新、表达、做出好东西 | Lego、Adobe | 创意工具/DIY/教育 |
| 统治者 (Ruler) | 权力、秩序、品质标杆 | Rolex、Mercedes | 高端/金融/商务 |

```markdown
## 品牌人格

### 主原型：{原型名}
为什么选它：{与品牌使命和用户期望的连接}

### 辅原型：{原型名}（可选）
加入辅原型的目的：{补充主原型缺少的维度}

### "如果品牌是一个人"
- 年龄感：{不是目标用户年龄，是品牌自身的"年龄感"}
- 穿着风格：{视觉隐喻}
- 说话方式：{直接/含蓄/幽默/严肃/温暖/酷}
- 社交场景中的角色：{领导者/好朋友/导师/同行者/守护者}
- 让人觉得：{用户和这个品牌在一起时的感受}

### 人格边界
- 我们是：{3-5 个形容词}
- 我们不是：{3-5 个要回避的形容词}
- 我们绝不：{底线行为}
```

## 模块 4：品牌语调体系（Tone of Voice）

语调不是"写文案用什么词"，是品牌在所有触点上说话的方式。

### 语调维度

```markdown
## 品牌语调体系

### 语调四维定位
| 维度 | 我们的位置 | 说明 |
|------|----------|------|
| 正式 ←→ 随意 | {如：偏随意，6/10} | {什么时候更正式，什么时候更随意} |
| 严肃 ←→ 幽默 | {如：中间偏幽默，7/10} | {哪些场景可以幽默，哪些不行} |
| 尊重 ←→ 不敬 | {如：尊重但不卑微，3/10} | {对用户平视，不跪舔也不居高} |
| 热情 ←→ 克制 | {如：温暖但不过度，4/10} | {表达关心但不煽情} |

### 语言偏好
- **常用词**：{品牌常用的词汇集合}
- **避免词**：{品牌绝不用的词}
- **句式偏好**：{短句干脆 / 长句有节奏 / 口语化 / 文艺感}
- **表情符号**：{用/不用，如果用，偏好哪些}
- **中英混用**：{是否允许，在什么场景}

### 不同场景的语调调整

| 场景 | 语调微调 | 示例 |
|------|---------|------|
| 产品描述 | {更专业，聚焦功能价值} | |
| 社媒日常 | {更轻松，像朋友聊天} | |
| 客诉回复 | {更温暖，先共情再解决} | |
| 品牌宣言 | {更有力量，简洁有节奏} | |
| 促销活动 | {更直接，但不low} | |
| 危机公关 | {更严肃，诚恳负责} | |

### 语调示例对照

| 场景 | ❌ 不像我们 | ✅ 像我们 |
|------|-----------|----------|
| 欢迎新用户 | | |
| 产品缺货 | | |
| 用户好评 | | |
| 被质疑时 | | |
```

## 模块 5：品牌视觉语言（Visual Identity Direction）

不做设计稿，给出清晰的品牌视觉方向定义，让 mc-aigc 和设计团队有据可依。

```markdown
## 品牌视觉方向

### 色彩系统
- 品牌主色：{色彩 + 情感含义}
- 辅助色：{色彩 + 使用场景}
- 禁用色：{品牌绝不使用的颜色}
- 色彩比例原则：{如 70% 主色 / 20% 辅色 / 10% 点缀}

### 视觉调性关键词
{5-7 个定义品牌视觉感受的词，供 mc-aigc 直接转化为 Prompt}
示例：clean / warm / organic / soft / luminous / human / minimal

### 视觉做与不做
| ✅ 做 | ❌ 不做 |
|-------|--------|
| {品牌视觉风格正面描述} | {品牌视觉风格反面描述} |

### 摄影/影像风格方向
- 光线偏好：{自然光/人工光/柔光/硬光}
- 构图偏好：{留白/紧凑/对称/自然随性}
- 人物呈现：{真实/理想化/场景化/特写}
- 场景偏好：{室内/户外/纯色背景/生活场景}

### 字体/排版方向
- 中文字体气质：{圆润亲切 / 锐利现代 / 经典宋体 / 手写感}
- 英文字体气质：{Sans-serif 现代 / Serif 经典 / Display 张力}
- 排版密度：{呼吸感强（大留白）/ 信息紧凑}
```

## 模块 6：品牌架构（Brand Architecture）

当品牌不止一个产品线时，需要回答"它们之间是什么关系"。

### 品牌架构模型

| 模型 | 逻辑 | 适用场景 | 例子 |
|------|------|---------|------|
| 统一品牌 (Branded House) | 所有产品共用一个品牌名 | 品牌力强，品类延展性好 | Apple / Google / 小米 |
| 背书品牌 (Endorsed) | 子品牌+母品牌背书 | 品类差异大但想借母品牌信任 | 万豪系列酒店 / Nestlé |
| 独立品牌 (House of Brands) | 每条产品线独立品牌 | 品类/人群差异大，不想互相影响 | P&G / 联合利华 |
| 混合模型 | 以上的组合 | 大多数成长期品牌 | |

```markdown
## 品牌架构

### 选定模型：{模型名}
理由：{为什么这种结构适合当前阶段和品类}

### 品牌层级
| 层级 | 品牌/产品线 | 定位 | 目标人群 | 价格带 |
|------|-----------|------|---------|--------|
| 母品牌 | | | | |
| 子品牌/产品线 1 | | | | |
| 子品牌/产品线 2 | | | | |

### 品牌关系规则
- 母品牌与子品牌的 logo 关系：{锁定/分离/灵活}
- 母品牌出现的场景：{所有触点 / 仅官网和包装背面 / 不出现}
- 子品牌的自由度：{独立视觉系统 / 在母品牌视觉内变体}
```

## 模块 7：品牌试金石（Brand Litmus Test）

一套快速判断"这个决定是否符合品牌"的检查清单。

```markdown
## 品牌试金石

当你不确定一个决策（文案/产品/合作/视觉）是否"品牌"时，过一遍这个清单：

### 5 秒判断
- [ ] 它像我们会做的事吗？
- [ ] 用户看到时，知道这是我们吗（即使没有 logo）？
- [ ] 它让我们的目标用户觉得"这个品牌懂我"吗？

### 深度判断
- [ ] 它符合我们的品牌使命吗？
- [ ] 它的语调在我们定义的范围内吗？
- [ ] 它的视觉在我们定义的范围内吗？
- [ ] 它不会让我们的用户觉得被冒犯/被俯视/被忽视吗？
- [ ] 如果竞品也做同样的事，我们做出来还有辨识度吗？

### 红线判断
- [ ] 它有没有违反我们"绝不"清单中的任何一条？
- [ ] 它有没有踩到文化雷区（参考 mc-insight 踩雷预警）？
```

## 输出模式

### 快速品牌梳理

用户说"帮我理一下品牌定位"时，输出：
- 定位公式
- 人格关键词
- 语调四维定位
- 控制在 1 页以内

### 完整品牌圣经

用户要"做一套完整的品牌策略"时，输出全部 7 个模块，形成 `brand.md`。

### 定向模块

用户只需要某个维度（"帮我定语调" / "品牌架构怎么搭" / "品牌人格选什么原型"），只输出对应模块。

## 与其他技能的联动

| 联动技能 | mc-brand 如何输入给它 |
|---------|---------------------|
| mc-storyteller | 品牌人格+使命+价值观 → 定义叙事基调和故事角色 |
| mc-insight | 品牌定位 → 在文化地图中找到匹配的表达领土 |
| mc-content | 语调体系+视觉方向 → 内容风格标准 |
| mc-aigc | 视觉语言关键词 → Prompt 基础层 |
| mc-product | 品牌承诺+价值观 → 产品定义的约束条件 |
| mc-kol | 品牌人格 → KOL 筛选标准（调性匹配） |
| mc-campaign | 品牌定位 → campaign 传播主张的边界 |
| mc-review | 品牌试金石 → 内容审查的品牌维度 |

**brand.md 是全局文件。mc-dispatch 会自动检测它的存在，在调用任何技能时注入品牌上下文。**

---

## 交互规则

- mc-brand 通常是新品牌/新 campaign 的第一步
- 产出 `brand.md` 后会持续被其他所有技能引用
- 品牌不是一成不变的：用户可以说"品牌升级"，系统在现有 brand.md 基础上迭代
- 输出完成后告诉用户文件路径，不要在对话中重复全文
